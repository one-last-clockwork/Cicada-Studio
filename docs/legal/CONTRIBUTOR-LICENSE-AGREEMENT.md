# Cicada Studio Contributor License Agreement

この文書は、Cicada Studio に貢献する人が同意する Contributor License Agreement の本文です。

## 1. 定義

**Project**：Cicada Studio、その関連リポジトリ、派生版、商用版、hosted 版、cloud 版、proprietary 版、closed source 版を指します。

**Contribution**：Project に提出されるコード、ドキュメント、テスト、翻訳、画像、アイコン、デザイン、テンプレート、設定、その他の著作権が発生しうる素材を指します。

**Contributor**：Contribution を提出する個人または組織を指します。

**Maintainer**：Project を管理し、Contribution を受け入れる権限を持つリポジトリ所有者、maintainer、またはその権利を承継した個人もしくは法人を指します。

## 2. 権利の許諾

Contributor は Maintainer に対し、Contribution を利用するための、非独占的、全世界的、無償、取消不能、永続的、サブライセンス可能、譲渡可能なライセンスを許諾します。

このライセンスには、Contribution を使用、複製、改変、翻案、結合、公開、配布、販売、提供、ホスト、再ライセンス、サブライセンスする権利を含みます。

このライセンスには、Contribution を AGPL-3.0-or-later、MIT、commercial license、proprietary license、closed source license、その他 Maintainer が選ぶライセンスで提供する権利を含みます。

このライセンスには、Contribution を無料版、有料版、Community 版、商用版、hosted 版、cloud 版、proprietary 版、closed source 版に含める権利を含みます。

## 3. 著作権の保持

Contributor は、Contribution について自分が持つ著作権を保持します。

この CLA は、Contributor から Maintainer への著作権譲渡ではありません。

Contributor は、自分の Contribution を別のプロジェクトや別のライセンスで利用できます。

## 4. 権限の表明

Contributor は、Contribution を提出する権利を持っていることを表明します。

Contributor は、Contribution が第三者の著作権、営業秘密、契約上の義務、雇用契約、発注契約、利用規約に違反しないことを確認します。

Contributor が勤務先または組織の業務として Contribution を提出する場合、Contributor はその組織から必要な権限を得ていることを確認します。

Maintainer は、組織由来の Contribution について corporate CLA または追加確認を求めます。

## 5. Third-Party Material

Contribution に third-party material が含まれる場合、Contributor はその出典、ライセンス、利用条件を pull request または関連文書で明示します。

Contributor は、Maintainer がその material を AGPL 版と proprietary 版の両方で利用できることを確認します。

Maintainer は、出典またはライセンスが不明な material を受け入れません。

## 6. 特許

Contributor は、Contribution を利用するために必要な範囲で、Contributor が保有または管理する特許請求項について、非独占的、全世界的、無償、取消不能、永続的な特許ライセンスを Maintainer と Project の利用者に許諾します。

この特許ライセンスは、Contribution そのものによって必然的に侵害される特許請求項に限ります。

## 7. 保証の否認

Contribution は現状有姿で提供されます。

Contributor は、適用法で許される範囲で、Contribution について明示または黙示の保証を行いません。

## 8. 受け入れの裁量

Maintainer は、Contribution を受け入れる義務を負いません。

Maintainer は、Contribution を受け入れる、変更する、分割する、書き直す、拒否する、削除することができます。

## 9. 同意方法

Contributor は、Maintainer が指定する方法でこの CLA に同意します。

自動 CLA チェックが導入されていない場合、pull request 上の明示的な同意コメントを同意記録として扱います。
